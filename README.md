## 关注方向

- **产品思维**：先判断问题，再决定方案；先讲价值，再谈功能
- **全栈落地**：能从产品设计走到前后端实现、接入、部署和运维
- **AI 工具产品化**：关注工具是否真的能进入工作流，而不只是 demo
- **Agent 系统**：关注任务分解、上下文组织、协作链路与执行稳定性
- **Memory System**：关注长期记忆、检索策略、状态管理与反馈闭环

  <sub>一个中文优先的 AI 输出修正工具（rewrite skill）</sub>
</p>

它要解决的，不是“把文字润得更华丽”，而是把 **模板腔、表演感、套话和不自然的 AI 味** 压下去，让表达更像真实的人在当前场景里会说的话。

这个项目之所以重要，不只是因为它能改写文本，而是因为它很能代表我的做事方式：

- 从真实问题出发，而不是从概念出发
- 把模糊体验拆成可执行规则，而不是停在感受层
- 在处理风格之前，先保护事实、术语、代码和技术上下文
- 让规则可以被复用、版本化、评测和持续迭代

## 正在推进

有两条线我在持续推进，但会保持克制表达，等抽象稳定后再逐步开源：

- **OpenClaw / AI infra & ops**  
  部署、接入、稳定性、运维、可维护性，以及系统真正跑起来之后的问题

- **Agent Memory System**  
  长期记忆、上下文组织、检索与协作能力，以及多轮任务中的信息延续

我不会在这里把未开源内容写成大而空的 roadmap。  
等边界稳定、能给出可复用形态时，再逐步放出来。

## 做事方式

- 先判断“为什么做”，再决定“怎么做”
- 先做最小闭环，再扩展复杂度
- 先保信息和语境，再谈风格和体验
- 先把东西做出来，再把它跑稳、跑久

## 一句原则

> 把模糊想法变成清晰方案，  
> 再把清晰方案做成可交付产品。

> Product sense decides **what to build**.  
> Engineering decides **whether it can actually ship**.
